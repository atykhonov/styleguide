--- extends: one_column_menu_left.j2 page_title: Chevah Style Guide --- This pages described the style guides / coding conventions used for the Chevah project. We hope that by following these guide you will find easier to collaborate with others and improve the quality of your code. There is a page with common rules and a special page for each language or technology used by the project. Please note that these guides are continuously updated according to latest best practices. I consider that arguments over these guides are not pointless. If you have any comment or suggestions, please get in touch with us. You can do that over email, or adding an issue or pull request on `chevah/styleguide - Github`_. We try to automatically check these rules using `pocket-lint`_. This is a collection or recommendations and ideas from various sources.
